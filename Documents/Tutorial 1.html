Resumen de Carga

Se cargaron 4,651 registros reales de inscritos desde unificado.xlsx para entrenar el modelo de la Universidad de Antioquia y la Corporación Lasallista.

1. ¿De dónde provienen nuestros datos?

Para este proyecto del concurso Datos al Ecosistema 2026, utilizamos dos fuentes primarias de información:

Datos Locales Reales (Excel)

Un archivo unificado.xlsx de 4,651 registros del comportamiento de inscripciones en programas de pregrado de la **Universidad de Antioquia** y la **Lasallista** entre los años 2014 y 2024.

Datos Abiertos de Colombia (API)

Conexión directa por HTTPS al portal **datos.gov.co** consumiendo endpoints JSON de microdatos de deserción regional, salarios promedio (OLE) e indicadores del DANE.

2. Conectándonos con Socrata (SODA API)

El portal de Datos Abiertos Colombia corre sobre la tecnología **Socrata**, la cual expone una API estructurada conocida como **SODA API**.

En el framework, la clase cliente ColombiaDatosAbiertosClient gestiona de forma profesional estas peticiones de manera asíncrona: 

# Cómo consulta nuestro código el dataset gubernamental
import requests
import pandas as pd

class ColombiaDatosAbiertosClient:
    BASE_URL = "https://www.datos.gov.co/resource"

    def fetch_dataset(self, dataset_id: str, limit: int = 1000) -> pd.DataFrame:
        url = f"{self.BASE_URL}/{dataset_id}.json"
        params = {"$limit": limit}
        headers = {"Accept": "application/json"}
        
        response = requests.get(url, headers=headers, params=params, timeout=10)
        if response.status_code == 200:
            return pd.DataFrame(response.json())
        return pd.DataFrame()

3. Cruce de Datos (ETL)

El proceso de **Extracción, Transformación y Carga (ETL)** lee simultáneamente el archivo Excel de inscritos reales y los parámetros demográficos.

4. Generación de Reglas de Deserción

Al no contar con una columna binaria explícita de deserción en los registros crudos de matrícula, el pipeline utiliza lógica de negocio para definir la variable objetivo (desertor):

Promedio < 3.0

Aumenta el riesgo académico en un +40%.

Materias Perdidas ≥ 2

Suma riesgo en materias perdidas en un +30%.

Estrato 1 y 2

Añade riesgo de vulnerabilidad socioeconómica (+20%).

5. Consumo en el Dashboard Interactivo

La aplicación Streamlit consume esta base de datos procesada para alimentar la visualización geográfica:

  • Métricas de Tarjetas: Muestra al instante el conteo total de estudiantes procesados (más de 22,000 registros unificados).
  • Mapa Coroplético/Burbujas: Lee las coordenadas geográficas de los estudiantes y colorea con un gradiente rojo las regiones con mayor riesgo de deserción histórica.
  • Simulador de Inferencia: Permite a los usuarios de bienestar universitario introducir variables reales de un estudiante para calcular su probabilidad clínica de abandono.
